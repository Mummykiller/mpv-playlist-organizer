# Project Architecture

This document provides a concise overview of the project's architecture, focusing on the roles and interactions of its core components.

## High-Level Overview

The project is a browser extension that uses native messaging to communicate with a Python backend. The frontend handles UI and playlist logic within the browser, while the backend manages the MPV media player process.

### Primary Technologies
-   **Frontend**: JavaScript (ES6 Modules), HTML, CSS
-   **Browser**: Manifest V3
-   **Backend**: Python 3.7+
-   **MPV Scripting**: Lua

### Core Interaction Flow (Example: Play Command)
1.  **UI (`content.js` / `popup.js`)**: User clicks "Play".
2.  **`background.js`**: The UI sends a `play` message to the background script.
3.  **`nativeConnection.js`**: The background script calls the native host via this module.
4.  **`native_host.py`**: Receives the `play` command, retrieves the playlist from `file_io.py`, and delegates to `mpv_session.py`.
5.  **`mpv_session.py`**: Starts or syncs with the MPV process, passing it the playlist URLs.
6.  **MPV**: The media player receives the URLs and begins playback.

### Operating Environment
-   **OS Compatibility**: The native host and installer are designed to be cross-platform, with specific logic for Windows, macOS, and Linux.
-   **Runtime Behavior**: The native host (`native_host.py`) is launched on-demand by the browser when the extension establishes a connection. It runs as a background process for the duration of the session and terminates when the connection is closed. It can also be invoked directly from the command line for CLI operations.

### Security Model
-   **Native Messaging Security**: Communication between the browser and the native host is secured by the browser's native messaging protocol. The manifest file, generated by `Installer.py`, explicitly declares the extension's ID. The browser ensures that only this specific extension can connect to the native host, preventing unauthorized access from other extensions or web pages.
-   **File Access**: The native host's file operations are restricted. For instance, file imports are sandboxed to the `exported` directory within the user's application data folder to prevent arbitrary file access.

### Error Handling & Resilience
-   **Native Host Unresponsiveness**: The `nativeConnection.js` module in the background script manages the connection port. If the native host crashes or becomes unresponsive, the port disconnects, causing pending requests to be rejected with an error. The extension's UI provides feedback to the user in such cases.
-   **MPV Process Failure**: The `mpv_session.py` module monitors the MPV process. If the process exits unexpectedly, it sends an `mpv_exited` message back to the browser with the exit code. This allows `background.js` to perform cleanup actions, such as clearing a playlist if the "Clear on Completion" setting is enabled.
-   **Playback Failures**: The native host monitors MPV's output for common `yt-dlp` failure patterns (e.g., HTTP 410 errors). Upon detection, it notifies the background script, which can then trigger a user prompt or an automatic update of `yt-dlp`, based on user preferences.

## Core Components

### Browser Extension (Frontend)

The frontend is a browser extension responsible for the user interface and interaction within the browser.

-   **`manifest.json`**: Defines the extension's structure, permissions, and entry points.
    -   **Interactions**:
        -   Declares `background.js` (service worker), `content.js` (content script), and `popup.html` (action popup).
        -   Requests `nativeMessaging` permission to communicate with `native_host.py`.
-   **`background.js`**: The service worker; acts as the central hub for state management and communication.
    -   **Imports/Dependencies**:
        -   `./utils/storageManager.js`
        -   `./utils/nativeConnection.js`
        -   `./utils/contextMenu.js`
        -   `./utils/playlistManager.js`
    -   **Interactions**:
        -   Communicates with `content.js` and `popup.js`.
        -   Sends commands to `native_host.py` via `nativeConnection.js`.
        -   Manages a sequential playback queue to process and send videos one-by-one to the native host.
        -   Handles MPV exit codes for playlist completion logic.
-   **`content.js`**: Injected into web pages to provide the on-page UI controller.
    -   **Imports/Dependencies**:
        -   Utilizes: `UIManager`, `PlaylistUI`, `AniListUI`, `PageScraper`, `Draggable`.
    -   **Interactions**:
        -   Two-way communication with `background.js`.
        -   Instantiates and coordinates UI modules (`UIManager`, `PlaylistUI`, `AniListUI`).
        -   Uses `content.css` for styling.
        -   Interacts with the page DOM for UI injection and SPA handling.
        -   Listens for global keyboard shortcuts defined in settings.
-   **`content.css`**: Stylesheet for the UI elements injected by `content.js`.
-   **`popup.html`**: Defines the HTML structure for the extension's popup window.
-   **`popup.js`**: Provides the logic for the popup window (folder/playlist management, settings).
    -   **Imports/Dependencies**:
        -   Utilizes: `UIModeManager` (defined internally), `OptionsManager` (from `settings.js`), `AniListRenderer` (from `utils/anilist_renderer.js`).
    -   **Interactions**:
        -   Manipulates the `popup.html` DOM.
        -   Two-way communication with `background.js`.
        -   Uses `settings.js` and `utils/anilist_renderer.js`.
-   **`popup.css`**: Stylesheet for the `popup.html` window.

### Native Host (Backend)

The backend is a Python application that acts as a bridge between the browser extension and the MPV media player.

-   **`native_host.py`**: The main entry point for the native messaging host and CLI.
    -   **Imports/Dependencies**:
        -   Local modules: `mpv_session.py`, `file_io.py`, `cli.py`, `services.py`.
    -   **Interactions**:
        -   Communicates with `background.js`.
        -   Delegates tasks to `mpv_session.py`, `file_io.py`, `services.py`, and `cli.py`.
        -   Handles an `append` command to add videos to the running MPV instance's playlist.
        -   Executes bypass scripts which can return JSON containing a resolved URL and necessary HTTP headers.
-   **`mpv_session.py`**: Manages the MPV player instance (start, stop, sync, IPC).
    -   **Imports/Dependencies**:
        -   Injected dependencies (from `native_host.py`): `is_process_alive`, `send_ipc_command`, `get_all_folders_from_file`, `get_mpv_executable`, `get_ipc_path`, `log_stream`, `send_message`, `SCRIPT_DIR`.
    -   **Interactions**:
        -   Manages the MPV process lifecycle and state.
        -   Persists session state to `session.json` within the user's application data directory.
-   **`services.py`**: Handles backend services like dependency checks (MPV, yt-dlp) and AniList data fetching.
    -   **Imports/Dependencies**:
        -   Standard Python modules.
    -   **Interactions**:
        -   Called by `native_host.py`.
        -   Executes `anilist_releases.py` as a subprocess.
        -   Runs system commands (e.g., `yt-dlp -U`).
-   **`file_io.py`**: Handles all filesystem interactions, managing key data files like `folders.json` and `config.json` within the user's application data directory.
    -   **Imports/Dependencies**:
        -   Standard Python modules.
    -   **Interactions**:
        -   Called by `native_host.py`, `mpv_session.py`, and `cli.py` for all file-based data access.
        -   Defines and manages paths within the platform-specific user data directory (e.g., `%APPDATA%` on Windows, `~/.local/share` on Linux).
-   **`cli.py`**: Implements the command-line interface logic (e.g., `play`, `list`).
    -   **Imports/Dependencies**:
        -   Injected dependencies (from `native_host.py`): `file_io` module, `mpv_session` module.
    -   **Interactions**:
        -   Invoked by `native_host.py` to handle CLI commands.
        -   Uses `file_io.py` and `mpv_session.py`.

### Supporting Scripts

-   **`Installer.py`**: GUI-based script to install/uninstall the native host.
    -   **Imports/Dependencies**:
        -   GUI modules: `tkinter` and its submodules (`ttk`, `scrolledtext`, `messagebox`, `filedialog`).
        -   Windows-specific: `winreg` (conditional).
    -   **Interactions**:
        -   Generates browser manifest files and wrapper scripts (`.bat`/`.sh`).
        -   Writes `data/config.json` with the MPV path.
        -   Dynamically generates `play_with_bypass.sh` (or `.bat`) based on user's browser selection.
        -   Performs dependency checks for `mpv` and `yt-dlp` on startup.
        -   Interacts with the OS (filesystem, Windows Registry).
-   **`on_completion.lua`**: A Lua script for MPV to detect when a playlist finishes.
    -   **Interactions**:
        -   Loaded by MPV when `mpv_session.py` enables the 'clear on completion' feature.
        -   **Exit Code (99)**: Causes MPV to exit with a specific code (99) upon natural playlist completion, which is then detected by the Python backend.
-   **`run_native_host.bat` / `run_native_host.sh`**: Wrapper scripts generated by `Installer.py` to launch `native_host.py` silently and with the correct environment.
-   **`play_with_bypass.sh` / `play_with_bypass.bat`**: A script generated by the installer to resolve URLs from specific sites (e.g., AnimePahe) that require special headers or cookies. It outputs a JSON object with the resolved URL and required HTTP headers for `native_host.py` to use.

### Utility Modules (`utils/`)

This directory contains helper modules for the frontend JavaScript.

-   **`anilist_renderer.js`**: Fetches AniList data (via `background.js`) and renders it into an HTML container. Used by `popup.js` and `AniListUI.js`.
-   **`AniListUI.js`**: Manages the in-page AniList side panel (state, rendering, events).
-   **`contextMenu.js`**: Creates and manages the browser's right-click context menus.
-   **`Draggable.js`**: Utility class to make an HTML element draggable.
-   **`Resizable.js`**: Utility class to make an HTML element resizable.
-   **`nativeConnection.js`**: Manages the persistent connection to the native Python host.
-   **`PageScraper.js`**: Extracts titles and URLs from web pages.
-   **`playlistManager.js`**: Centralizes all playlist modification logic (add, remove, clear, reorder).
-   **`PlaylistUI.js`**: Manages rendering and interaction for the playlist view in the on-page UI.
-   **`settings.js`**: Manages the settings UI within the popup, including the tag-based MPV flags input.
-   **`storageManager.js`**: Manages all `chrome.storage.local` operations. This includes persisting core application data (folders, folder order) and all user preferences. It also handles data structure migrations between extension versions.
-   **`UIManager.js`**: Manages the creation, injection, and teardown of all on-page UI elements.
